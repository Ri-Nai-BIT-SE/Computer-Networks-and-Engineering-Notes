# 1.7.2 协议与划分层次

## 网络协议三要素

### 语法（Syntax）
- **定义**：数据与控制信息的结构或格式
- **说明**：规定通信双方"如何讲"

### 语义（Semantics）
- **定义**：需要发出何种控制信息，完成何种动作以及做出何种响应
- **说明**：规定通信双方"讲什么"

### 同步（Synchronization）
- **定义**：事件实现顺序的详细说明
- **说明**：规定通信双方"何时讲"

## 分层的原则
1. 各层之间相互独立
2. 每层只实现一种相对独立的功能
3. 层与层之间通过接口进行交互
4. 下层为上层提供服务
5. 每层的功能选择应有利于标准化

## 分层的优缺点

### 优点
- 各层之间相互独立
- 灵活性好
- 结构上可分割开
- 易于实现和维护
- 能促进标准化工作

### 缺点
- 有些功能可能在不同层重复出现
- 层数过多会导致系统效率降低
