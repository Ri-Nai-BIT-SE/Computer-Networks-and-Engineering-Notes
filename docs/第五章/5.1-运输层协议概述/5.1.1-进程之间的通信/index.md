# 5.1.1 进程之间的通信

## 运输层的作用

- **向上**：为应用层提供通信服务
- **向下**：利用网络层提供的服务
- **核心定义**：
  - 网络层负责**主机之间**（Host-to-Host）的通信
  - 运输层负责**应用进程之间**（Process-to-Process）的通信
- **端到端通信**：运输层是整个网络体系结构中，从下往上数第一个提供**端到端**（End-to-End）服务的层次

## 逻辑通信

- 运输层向高层用户屏蔽了下面网络核心的细节（如网络拓扑、路由选择等）
- 应用进程看来，好像两个运输层实体之间有一条**水平的、端到端的逻辑通信信道**

## 复用 (Multiplexing) 与 分用 (Demultiplexing)

- **复用**：发送方不同的应用进程都可以使用同一个运输层协议传送数据（都需要加上头部）
- **分用**：接收方的运输层在剥去报文首部后，能够把这些数据正确交付给目的应用进程
- *注：这是通过**端口号**来实现的*

## 差错检测

- 网络层（IP数据报）主要检验首部，而运输层会对**数据部分**和**首部**进行差错检测
