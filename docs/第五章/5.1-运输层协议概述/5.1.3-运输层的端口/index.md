# 5.1.3 运输层的端口 (Port)

为了区分同一台主机上的不同应用进程，使用**端口号**（Protocol Port Number）作为标志。

## 端口号的性质

- **软件端口**：不同于路由器或交换机的硬件端口，它是协议栈层间的抽象接口
- **长度**：**16位** 二进制
- **取值范围**：**0 ~ 65535**
- **本地意义**：端口号只在本地计算机上有意义，不同计算机的相同端口号没有联系

## 端口的三种分类 (★常考选择题)

| 分类 | 端口号范围 | 描述 |
| :--- | :--- | :--- |
| **熟知端口 (Well-known Ports)** | **0 ~ 1023** | 指派给TCP/IP最重要的一些应用程序，由IANA分配 |
| **登记端口 (Registered Ports)** | **1024 ~ 49151** | 为没有熟知端口的应用程序使用，需在IANA登记 |
| **客户端口 / 短暂端口** | **49152 ~ 65535** | 客户进程运行时动态选择，通信结束后系统回收 |

## 常用的熟知端口 (★必须背诵)

- **FTP (文件传输)**: 21 (控制连接) / 20 (数据连接)
- **TELNET (远程终端)**: 23
- **SMTP (邮件发送)**: 25
- **DNS (域名解析)**: 53 (UDP/TCP均有，主要用UDP)
- **TFTP (简单文件传输)**: 69 (UDP)
- **HTTP (万维网)**: 80
- **SNMP (网管)**: 161
- **HTTPS**: 443

## 套接字 (Socket)

- TCP连接的端点叫做套接字
- **Socket = (IP地址 : 端口号)**
- 每一条TCP连接唯一地被通信两端的两个套接字所确定

---

## 🎓 考试重点总结

### 端口号

- **选择题考点**：熟知端口号的范围（0-1023）以及常用协议对应的端口号（如HTTP是80，DNS是53）
- 理解**复用**和**分用**是基于端口号实现的

### 通信层级

- 明确运输层是**进程到进程**（端到端）的通信，而网络层是主机到主机的通信
