# 3.4 可靠传输机制

数据链路层的主要目标之一是提供可靠的传输服务，即将网络层的数据报无差错地从一个节点传输到相邻节点。然而，底层的物理信道是不可靠的，可能会出现**比特差错**和**帧丢失**。此外，发送方和接收方的处理速率不匹配可能导致数据淹没接收方。

### 核心问题

设计可靠传输协议需要解决两个核心问题：

1.  **差错控制 (Error Control)**: 如何从比特差错和帧丢失中恢复。
2.  **流量控制 (Flow Control)**: 如何协调发送速率以匹配接收方的处理能力。

本节将通过一系列逐步复杂的协议来探讨如何实现可靠传输，这些协议统称为**自动重传请求 (ARQ - Automatic Repeat reQuest)**。
