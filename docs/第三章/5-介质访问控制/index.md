# 3.5 介质访问控制 (Medium Access Control, MAC)

### 核心概念

#### 1. 问题背景：共享信道

*   在许多网络中（尤其是局域网），多个节点连接到一个**共享的广播信道**上。这意味着任何一个节点发送的信号都会被所有其他节点接收。

*   **竞争与冲突 (Collision)**: 
    *   如果两个或两个以上的节点同时在共享信道上发送数据。
    *   它们的信号会相互干扰，导致传输失败。
    *   这种现象称为**冲突**或**碰撞**。

*   **关键问题**: 
    *   如何设计一个协议来协调多个节点对共享信道的访问。
    *   以公平、高效地分配信道资源，并解决冲突问题。
    *   这就是**介质访问控制 (MAC)** 要解决的问题。

#### 2. MAC 协议与 ARQ 协议的区别

*   **ARQ 协议 (如停-等, GBN, SR)**:
    *   **重心**: 解决**两个对等实体**之间因信道差错（丢帧、比特错误）导致的可靠传输问题。
    *   **参与者**: 仅涉及通信的双方。
    *   **关注点**: 差错控制、流量控制。

*   **MAC 协议**:
    *   **重心**: 解决广播式网络中，因**其他用户**的干扰（即冲突）而导致的信道访问问题。
    *   **参与者**: 需要网络中**所有**实体进行协调。
    *   **关注点**: 信道分配、冲突避免/检测。

#### 3. 信道分配的两种策略

1.  **静态划分信道**: 
    *   在用户通信之前，就预先、固定地将信道资源分配给每个用户。
    *   如：频分多路复用 (FDM)、时分多路复用 (TDM)。

2.  **动态媒体接入控制 (多点接入)**: 
    *   不预先分配资源，所有用户动态地竞争信道。
    *   这是 MAC 协议研究的重点。
