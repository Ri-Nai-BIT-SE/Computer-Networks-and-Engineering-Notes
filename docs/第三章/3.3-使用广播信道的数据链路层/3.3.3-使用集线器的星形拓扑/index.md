# 3.3.3 使用集线器的星形拓扑

## 集线器（Hub）

### 定义
- 多端口的转发器
- 工作在物理层

### 功能
- 将收到的信号进行整形放大后，向所有其他端口转发
- 不进行碰撞检测

### 特点
- **物理层设备**：只对信号进行放大和转发
- **逻辑上仍是总线网**：所有站点共享同一个总线
- **碰撞域**：使用集线器的所有站点都在同一个碰撞域内

## 星形拓扑

### 结构
- 所有站点通过双绞线连接到集线器
- 集线器位于中心

### 优点
- 便于维护和管理
- 故障隔离（某个站点故障不影响其他站点）
- 布线方便

### 缺点
- 集线器成为单点故障
- 逻辑上仍是总线网，存在碰撞问题

## 与总线型网络的比较

| 特性 | 总线型 | 星形（集线器） |
|------|--------|---------------|
| 物理拓扑 | 总线 | 星形 |
| 逻辑拓扑 | 总线 | 总线 |
| 碰撞域 | 整个网络 | 整个网络 |
| 设备 | 无 | 集线器（物理层） |

## 集线器 vs 交换机（重要对比）⭐ **必考概念辨析**

这是本章最核心的设备对比，常结合**网络拓扑图**考查。

| 特性 | 集线器 (Hub) | 交换机 (Switch) |
|:---|:---|:---|
| **工作层次** | 物理层 | 数据链路层 |
| **带宽** | **共享带宽**（所有口平分总带宽） | **独占带宽**（每个口都是全速） |
| **碰撞域** | 所有接口在**同一个**碰撞域 | **每个接口**是一个独立的碰撞域 |
| **广播域** | 同一个广播域 | 同一个广播域（除非划分VLAN） |
| **工作模式** | 半双工（使用CSMA/CD） | 全双工（不需要CSMA/CD） |
| **转发方式** | 广播到所有端口 | 根据MAC地址表点对点转发或广播 |
| **总吞吐量** | 受限于单个端口速率 | $N$个端口的总容量 = $N \times \text{单端口速率}$ |

### 典型考题 ⭐ **必考题型**
1. **碰撞域计数**：给出拓扑图，数一数有多少个碰撞域？
   - 集线器：不隔离碰撞域，所有连接的设备在同一个碰撞域
   - 交换机：每个端口是一个独立的碰撞域

2. **总吞吐量计算**：
   - $N$个用户连在10Mb/s集线器上：总容量 = **10Mb/s**（共享）
   - $N$个用户连在10Mb/s交换机上：总容量 = **$N \times 10$Mb/s**（独占）
