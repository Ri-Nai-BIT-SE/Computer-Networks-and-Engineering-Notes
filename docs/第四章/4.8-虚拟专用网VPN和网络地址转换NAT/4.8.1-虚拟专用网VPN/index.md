# 4.8.1 虚拟专用网 VPN (Virtual Private Network)

## 1. 专用地址 (Private Addresses) / 本地地址

由于IP地址紧缺，且很多机构内部通信不需要接入互联网，RFC 1918 指明了一些**专用地址**，仅在机构内部使用，互联网路由器对这些地址**不进行转发**（即在公网上不可路由）。

### 三个专用地址块 ⭐⭐⭐ **必背考点**

- **A类**：`10.0.0.0` 到 `10.255.255.255` (即 **10.0.0.0/8**)
- **B类**：`172.16.0.0` 到 `172.31.255.255` (即 **172.16.0.0/12**)
- **C类**：`192.168.0.0` 到 `192.168.255.255` (即 **192.168.0.0/16**)

## 2. VPN 的原理

- **定义**：利用**公用互联网**作为本机构各专用网之间的通信载体，构建的**虚拟**专用网络。
- **核心技术**：隧道技术 (Tunneling)。
- **工作过程**：
  1. 机构A的路由器 R1 收到内部发往机构B（内网IP）的数据报。
  2. R1 将该数据报进行**加密**，并**封装**在一个新的 IP 数据报中（作为数据部分）。
     - *新数据报的源IP：R1的全球IP；目的IP：R2的全球IP。*
  3. 新数据报在公网上传输到达机构B的路由器 R2。
  4. R2 收到后，剥去外层首部，解密内层数据报，恢复出原始数据报（目的IP为机构B的内网IP），并交付给目的主机。
